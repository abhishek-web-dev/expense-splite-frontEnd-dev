Problem Statement

This is Expense Splitter Web Application. Which provide a platform, where user can create a group and they can add their friends. inside a group anyone can create a expense and this application will distribute equal amount to others. group members can comunicate with each others. for each operation users will get real-time notification and email messages.

Proposed system

This web application has these many features:

Login view

  1. User will be able to login to the application through his email and password.
  2. User will be able to create a new account on this platform.
  3. If user will forget his password, he can change his password using Forgot password feature.
  4. Once user will login, redirect to his personalized dashboard view. where his all gouups will be listed out.

User home page view

Here, user all groups name will be listed out and user can do these things -

  1. He can see all groups name.
  2. He can see who has created particular group.
  3. He can see when group has created.
  4. There is one button for creation of new group.
  5. User can select any number of users at the time of group creation.
  6. Once user will click any group he can see all expenses related to that group.

Group Details view

  1. Here user will be able to edit/delete Group.
  2. Here user can see pending amount of each person in this group.
  3. Any user can add any issue in chat group section.
  4. User can see all group chat messages.
  5. There is one button for creation of new expense.
  6. Here user can see all expenses which had created in that group.
  7. User can see each expense name, who has created and when had created.
  8. Once user will click any expense he will redirect to detail view of that expense.

Create New Expense

User will enter these information -

  1. Expense name
  2. Expense amount
  3. Expense payed by -
    1. Ypu : All expense amount has paid by expense creator.
    2. Multiple : Here expense creator will enter each person contribution amount.
    3. If expense creator want to remove someone at the time of expense creation, he can remove using remove button.
  4. Expense distributed -
    1. Equally : Total expense amount will distribute equally between all member of that group.
    2. Unequal : Here total expense amount will distribute only selected member.
    3. At the time of expense creation, expense creator can select each person name, who will pay for that expense.

Expense Details view

  1. Here user will be able to see -
    1. Expense name
    2. Expense amount
    3. Expense creator name
    4. Expense created date
    5. Expense last updated date
    6. Expense last updated person name
    7. How much money each person has contributed
    8. In how many people total amount has distributed
    9. Here user can see pending amount of each person in this expense.
  2. User can go to previous page.
  3. User can edit/delete that expense.
  4. If user will perform edit/delete operation each member of that expense will get real-time notification and email messages.

Expense History

  1. User can see history of that expense.
  2. While someone will edit the expense one detail history will be create like what changes has done and who has done these changes.
  3. All online user of that expense will get real-time notification if anyone edit the expense.
  4. Each user of that expense will get email if anyone edit the expense.
  5. User can see when expense history had created.
